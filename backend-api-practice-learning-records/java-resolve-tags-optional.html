Java学習ノート: resolveTags と Optional

対象コード

private List<Tag> resolveTags(UUID userId, List<String> tagNames) {
  if (tagNames == null || tagNames.isEmpty()) {
    return List.of();
  }
  LinkedHashSet<String> normalizedNames = new LinkedHashSet<>();
  for (String tagName : tagNames) {
    String normalizedName = normalizeBlank(tagName);
    if (normalizedName != null) {
      normalizedNames.add(normalizedName);
    }
  }
  return normalizedNames.stream()
      .map(name -> tagRepository.findByNameAndUserId(name, userId)
          .orElseGet(() -> tagRepository.save(Tag.create(userId, name))))
      .toList();
}

このメソッドは、タグ名の一覧を受け取り、未入力値を除外し、重複をなくし、 既存タグを取得または新規作成して List<Tag> として返します。

メソッド宣言

コード 意味
private 同じクラスの中からだけ呼び出せるメソッドであることを表します。
List<Tag> 戻り値の型です。Tag オブジェクトのリストを返します。
UUID userId 対象ユーザーを識別する ID です。
List<String> tagNames タグ名の文字列リストです。

null または空リストの判定

if (tagNames == null || tagNames.isEmpty()) {
  return List.of();
}
条件 意味
tagNames == null リスト自体が存在しない状態です。
tagNames.isEmpty() リストは存在するが、要素数が 0 の状態です。
List.of() 空のリストを作って返します。

ここでは、タグ名が渡されていない場合に処理を続けず、空の List<Tag> を返します。

LinkedHashSet の役割

LinkedHashSet<String> normalizedNames = new LinkedHashSet<>();

LinkedHashSet は、重複を許さず、追加した順番を保持するコレクションです。 タグ名の重複を取り除きつつ、ユーザーが入力した順番を保つために使われています。

入力 LinkedHashSet に追加後
["Java", "Spring", "Java"] ["Java", "Spring"]

拡張 for 文と正規化

for (String tagName : tagNames) {
  String normalizedName = normalizeBlank(tagName);
  if (normalizedName != null) {
    normalizedNames.add(normalizedName);
  }
}

for (String tagName : tagNames) は、tagNames の要素を 1 件ずつ tagName に取り出して繰り返す書き方です。

normalizeBlank(tagName) により、null、空文字、空白だけの文字列を null にそろえ、値がある場合は前後の空白を削ります。 その結果が null でないものだけを normalizedNames に追加します。

Stream と map

return normalizedNames.stream()
    .map(name -> tagRepository.findByNameAndUserId(name, userId)
        .orElseGet(() -> tagRepository.save(Tag.create(userId, name))))
    .toList();
コード 意味
normalizedNames.stream() タグ名の集合を Stream に変換します。
.map(name -> ...) 各タグ名 name を別の値に変換します。ここでは String を Tag に変換します。
.toList() Stream の処理結果を List に戻します。

map() の詳しい仕組み

map() は、Stream の中に流れている要素を 1 件ずつ受け取り、 別の値に変換して次の Stream に流すメソッドです。 今回は String のタグ名を Tag オブジェクトに変換しています。 

normalizedNames.stream()
    .map(name -> ...)
    .toList();
段階 Stream の中身 型のイメージ
normalizedNames ["Java", "Spring"] LinkedHashSet<String>
stream() 後 "Java"、"Spring" が 1 件ずつ流れる Stream<String>
map(...) 後 Tag(Java)、Tag(Spring) に変換される Stream<Tag>
toList() 後 変換後の Tag をリストにまとめる List<Tag>

map(name -> ...) の name には、 Stream から取り出されたタグ名が 1 件ずつ入ります。 たとえば最初の処理では name = "Java"、 次の処理では name = "Spring" というイメージです。

map() の中で Optional が Tag になる流れ

疑問になりやすい点は、次の部分です。 

.map(name -> tagRepository.findByNameAndUserId(name, userId)
    .orElseGet(() -> tagRepository.save(Tag.create(userId, name))))

tagRepository.findByNameAndUserId(name, userId) だけを見ると、 これは Optional<Tag> を返す処理です。 しかし、map() が最終的に受け取っている戻り値は Optional<Tag> ではなく Tag です。

理由は、findByNameAndUserId(...) の戻り値に対して、 すぐ後ろで .orElseGet(...) を呼び出しているからです。 

Optional<Tag> optionalTag =
    tagRepository.findByNameAndUserId(name, userId);

Tag tag =
    optionalTag.orElseGet(() -> tagRepository.save(Tag.create(userId, name)));

つまり、ラムダ式の中では次の 2 段階の処理が行われています。

処理 戻り値の型 意味
findByNameAndUserId(name, userId) Optional<Tag> 既存タグがあるかもしれない、ないかもしれない状態を返します。
.orElseGet(...) Tag 中身があればその Tag、なければ新規作成して保存した Tag を返します。
map(name -> ...) 全体 Tag 各 String のタグ名を、最終的に Tag へ変換します。

map() の戻り値の型は、ラムダ式の右側が最終的に返す型で決まります。 今回の右側は findByNameAndUserId(...) で終わっておらず、 .orElseGet(...) まで実行した結果を返しているため、 Stream<Optional<Tag>> ではなく Stream<Tag> になります。

もし orElseGet() がなかったら

仮に orElseGet() を書かず、次のようにした場合を考えます。 

normalizedNames.stream()
    .map(name -> tagRepository.findByNameAndUserId(name, userId))
    .toList();

この場合、map() の右側が返すのは Optional<Tag> です。 そのため、全体の結果は List<Tag> ではなく List<Optional<Tag>> になります。

コード map() の右側の戻り値 toList() 後の型
.map(name -> findByNameAndUserId(...)) Optional<Tag> List<Optional<Tag>>
.map(name -> findByNameAndUserId(...).orElseGet(...)) Tag List<Tag>

現在のコードでは、orElseGet() によって 「見つかった Tag」または「新しく保存した Tag」のどちらかが必ず得られるため、 map() は各タグ名を Tag に変換できます。

Optional とは

Optional は、値が存在するかもしれないし、存在しないかもしれないことを表す Java の型です。 たとえば Optional<Tag> は、「Tag が入っているかもしれない箱」と考えると理解しやすいです。

状態 例 意味
値がある Optional.of(tag) Tag が見つかった状態です。
値がない Optional.empty() 該当する Tag が存在しない状態です。

今回の tagRepository.findByNameAndUserId(name, userId) は、 既存タグが見つかれば Optional<Tag> に入れて返し、 見つからなければ空の Optional を返す処理だと考えられます。

orElseGet() の意味

tagRepository.findByNameAndUserId(name, userId)
    .orElseGet(() -> tagRepository.save(Tag.create(userId, name)))

orElseGet() は Optional のメソッドです。 Optional の中に値があればその値を返し、空の場合だけ引数の処理を実行して値を作ります。

Optional の状態 orElseGet() の動き
既存タグが見つかった その Tag を返します。新規保存処理は実行されません。
既存タグが見つからなかった Tag.create(userId, name) で作成し、tagRepository.save(...) で保存した結果を返します。

if 文で書いた場合のイメージ

orElseGet() を使った処理を、イメージしやすいように if 文で書くと次のようになります。 

Optional<Tag> optionalTag = tagRepository.findByNameAndUserId(name, userId);

Tag tag;
if (optionalTag.isPresent()) {
  tag = optionalTag.get();
} else {
  tag = tagRepository.save(Tag.create(userId, name));
}

実際のコードでは、この流れを orElseGet() によって短く表現しています。

QAまとめ

質問 答え
Optional とは何か 値があるかもしれないし、ないかもしれないことを表す型です。null を直接返すより、値の有無を明示できます。
orElseGet() は Optional 専用か はい。orElseGet() は Optional クラスのメソッドです。通常の String、List、Tag にはありません。
orElseGet() の右側はいつ実行されるか Optional が空のときだけ実行されます。今回なら、既存タグが見つからなかった場合だけ新規作成と保存が行われます。
map() は何をしているのか Stream の要素を 1 件ずつ受け取り、ラムダ式の結果に変換します。今回なら String のタグ名を Tag に変換しています。
findByNameAndUserId(...) は Optional を返すのに、なぜ map() の結果は Tag になるのか findByNameAndUserId(...) の直後に .orElseGet(...) を呼んでいるためです。orElseGet() が Optional<Tag> から最終的な Tag を取り出す、または作成して返します。

今回理解できた6点の整理

今回のコードを理解するうえでは、次の 6 点に分けて考えると整理しやすくなります。

項目 理解 今回のコードでの見方
1. Function とは何か Function<T, R> は、T 型の値を受け取り、 R 型の値を返す関数型インターフェースです。 中心となるメソッドは R apply(T value) です。 今回は String のタグ名を受け取り、Tag を返すため、 Function<String, Tag> と考えられます。
2. map() は Function を引数に取る map() の定義は <R> Stream<R> map(Function<? super T, ? extends R> mapper) です。つまり、値を変換するための Function を受け取ります。 .map(name -> ...) のラムダ式が、 Function として map() に渡されています。
3. stream().map() は要素を apply() に渡す map() は Stream の要素を 1 件ずつ取り出し、 渡された Function の apply() メソッドの引数として渡します。 normalizedNames が ["Java", "Spring"] なら、 内部的には mapper.apply("Java")、 mapper.apply("Spring") のように呼ばれるイメージです。 その引数がラムダ式の name です。
4. map() は処理後の値を要素にした新しい Stream を返す map() は、受け取った値に処理を行い、 その戻り値を新しい Stream の要素にします。 「処理した値そのものを 1 個返す」のではなく、 「処理した値たちを要素にした新しい Stream」を返します。 今回は Stream<String> の各要素を Tag に変換するため、 map() 後は Stream<Tag> になります。 最後に toList() で List<Tag> になります。
5. orElseGet() は Optional のメソッド orElseGet() は Optional に対して呼び出すメソッドです。 Optional が値を持っていればその値を返し、 空なら引数に渡された Supplier を実行します。 findByNameAndUserId(...) が返す Optional<Tag> に対して .orElseGet(...) を呼び出しています。
6. orElseGet() の戻り値の型は Optional が持つ型に制限される Optional<Tag> に対して orElseGet() を呼ぶ場合、 戻り値は Tag です。 空の場合に実行される Supplier も、 Tag または Tag として扱える型を返す必要があります。 tagRepository.save(Tag.create(userId, name)) は Tag を返します。 そのため Optional<Tag>.orElseGet(...) の戻り値として成立します。

この 6 点をまとめると、今回の map() は String を受け取って Tag を返す Function を使っており、 その結果として Stream<String> が Stream<Tag> に変換されている、と理解できます。

最終的に学んだこと

このコードでは、タグ名の入力を正規化し、重複を除去したうえで、 Optional と orElseGet() を使って 「既存タグがあれば使う、なければ作る」という処理を簡潔に表現しています。 map() はその処理をタグ名 1 件ずつに適用し、 最終的に Stream<String> を Stream<Tag> に変換しています。
