1. エンティティ設計とDBの関係

今回のアプリでは、ユーザーが会話に対してメッセージを送り、その会話に紐づいてAIの回答が保存される構造を想定しています。 

Conversation 1件
  ├─ UserMessage 複数
  └─ AiMessage 複数

DBテーブルとしては、概念的に次のようになります。

users
- id
- email
- name
- hash_password

conversation
- id
- ai_model
- created_at

user_messages
- id
- text
- user_id          -> users.id
- conversation_id  -> conversation.id
- created_at

ai_messages
- id
- text
- ai_model
- conversation_id  -> conversation.id
- created_at

JPAでは、Javaのフィールドにアノテーションを付けることで、DB上のカラムや外部キーとの対応を表します。 特にエンティティ同士の関連は @Column ではなく、@ManyToOne や @OneToMany で表します。

2. @ManyToOne の考え方

@ManyToOne は「多くの子レコードが、1つの親レコードに属する」関係を表します。 例えば、多くの UserMessage は1つの Conversation に属します。 

@ManyToOne(fetch = FetchType.LAZY, optional = false)
@JoinColumn(name = "conversation_id", nullable = false)
private Conversation conversation;

このコードの意味は次の通りです。

記述 意味
@ManyToOne このエンティティの多くの行が、相手エンティティの1行に紐づく。
fetch = FetchType.LAZY 関連先を必要になるまで読み込まない。
optional = false Java/JPA上、この関連をnull不可として扱う。
@JoinColumn(name = "conversation_id") このテーブルに作る外部キー列の名前を指定する。
nullable = false DB上、この外部キー列をNULL不可にする。

@ManyToOne を書く側が、基本的に外部キーを持つ側です。 したがって、UserMessage 側に conversation_id ができます。

「外部キーを持つ側 = @ManyToOne + @JoinColumn」と覚えると理解しやすいです。

3. @OneToMany(mappedBy = "...") の考え方

@OneToMany は、親側から子の一覧を参照したいときに使います。 例えば、Conversation からその会話に属する UserMessage 一覧を見たい場合です。 

@OneToMany(mappedBy = "conversation")
private List<UserMessage> userMessages = new ArrayList<>();

mappedBy = "conversation" の conversation はDBカラム名ではありません。 UserMessage クラスのJavaフィールド名です。 

// UserMessage.java
@ManyToOne(fetch = FetchType.LAZY, optional = false)
@JoinColumn(name = "conversation_id", nullable = false)
private Conversation conversation;

つまり mappedBy = "conversation" は、 「この関連の本体は UserMessage.conversation 側にあります」という意味です。

mappedBy = "conversation_id" ではありません。 conversation_id はDBカラム名であり、mappedBy に指定するのはJavaのフィールド名です。

所有者という考え方

JPAの双方向関連では、どちらか一方が「関連の所有者」になります。 今回の所有者は、外部キーを持つ UserMessage 側です。 

// 所有者側
@ManyToOne
@JoinColumn(name = "conversation_id")
private Conversation conversation;

Conversation 側の @OneToMany(mappedBy = "conversation") は、 逆方向から一覧を見られるようにするための定義です。

4. @Column が必要な場所・不要な場所

@Column は、単一の値をDBカラムに保存するためのアノテーションです。 

@Column(nullable = false, name = "text")
private String text;

@Column(name = "created_at")
private LocalDateTime createdAt;

一方、List<UserMessage> のような関連一覧は、DBの1カラムに保存される値ではありません。 

// NG
@OneToMany(mappedBy = "conversation")
@Column(nullable = false, name = "user_messages")
private List<UserMessage> userMessages;

Conversation テーブルに user_messages というカラムが作られて、そこに配列が入るわけではありません。 実際の関連は、子テーブル側の conversation_id によって表されます。 

// OK
@OneToMany(mappedBy = "conversation")
private List<UserMessage> userMessages = new ArrayList<>();

5. fetch = FetchType.LAZY

fetch = FetchType.LAZY は、関連先のエンティティを「必要になるまで読み込まない」設定です。 

@ManyToOne(fetch = FetchType.LAZY, optional = false)
@JoinColumn(name = "conversation_id", nullable = false)
private Conversation conversation;

例えば、次のように UserMessage を取得したとします。

UserMessage message = userMessageRepository.findById(1L).orElseThrow();

LAZY の場合、この時点では基本的に UserMessage 本体だけを読み込みます。 message.getConversation() などで関連先が必要になったタイミングで、Conversation を読みに行きます。

@ManyToOne のデフォルトは EAGER です。 つまり、明示しないと関連先を最初から読み込む動きになりがちです。 不要なSQLや意図しない読み込みを避けるため、@ManyToOne では LAZY を明示することが多いです。

6. コンストラクタ、getter、setterの設計

JPA用の引数なしコンストラクタ

JPAエンティティには、protected または public の引数なしコンストラクタが必要です。 アプリ側から自由に使わせる必要はないため、通常は protected にします。 

protected UserMessage() {
}

idのsetterは基本不要

id はDBが採番する値なので、外部から変更できるsetterは基本的に不要です。 

@Id
@GeneratedValue(strategy = GenerationType.IDENTITY)
private Long id;

public Long getId() {
    return id;
}

userやconversationのsetterも基本不要

「あるメッセージを書いたユーザー」や「あるメッセージが属する会話」は、作成後に変わると不自然です。 そのため、コンストラクタでセットし、setterは置かない設計が安全です。 

public UserMessage(String text, User user, Conversation conversation) {
    this.text = text;
    this.user = user;
    this.conversation = conversation;
    this.createdAt = LocalDateTime.now();
}

本文編集は意味のあるメソッド名にする

本文は編集機能を想定するなら変更可能にしてよいですが、単純な setText より editText のようなドメイン上の意味が分かる名前にすると読みやすくなります。 

public void editText(String text) {
    this.text = text;
}

7. enum、DTO、Validation

enumをDBに保存する

JPAでenumをDBに保存する場合は、@Enumerated(EnumType.STRING) を使います。 

public enum AiModel {
    GPT,
    GEMINI,
    CLAUDE
}
@Enumerated(EnumType.STRING)
@Column(name = "ai_model", nullable = false)
private AiModel aiModel;

EnumType.STRING にすると、DBには GPT、GEMINI、CLAUDE のような文字列で保存されます。 EnumType.ORDINAL は順番の数値で保存され、enumの並び替えに弱いため避けるのが無難です。

DTOでenumの値を制限する

リクエストDTOの型を AiModel にすると、JSONで受け取れる値はenumの中身に制限されます。 

public record CreateConversationRequest(
        @NotNull
        AiModel aiModel
) {
}

次はOKです。

{
  "aiModel": "GPT"
}

次は AiModel に存在しないためNGです。

{
  "aiModel": "CHATGPT"
}

@NotBlank は文字列用です。enumやLongなどには @NotNull を使います。

Controllerでは @Valid を付けることでBean Validationを有効にします。

@PostMapping("/conversations")
public ResponseEntity<CreateConversationResponse> createConversation(
        @Valid @RequestBody CreateConversationRequest request
) {
    return ResponseEntity.status(HttpStatus.CREATED).body(chatService.createConversation(request));
}

8. Service層で投げる例外

sendMessage では、userId と conversationId からDBを検索します。 存在しないIDが指定された場合は、404 Not Found系の例外にするのが自然です。 

User messageSender = userRepository.findById(request.userId())
        .orElseThrow(() -> new UserNotFoundException(request.userId()));

Conversation targetConversation = conversationRepository.findById(request.conversationId())
        .orElseThrow(() -> new ConversationNotFoundException(request.conversationId()));

例外クラス例です。

@ResponseStatus(HttpStatus.NOT_FOUND)
public class UserNotFoundException extends RuntimeException {

    public UserNotFoundException(Long userId) {
        super("User not found: " + userId);
    }
}
@ResponseStatus(HttpStatus.NOT_FOUND)
public class ConversationNotFoundException extends RuntimeException {

    public ConversationNotFoundException(Long conversationId) {
        super("Conversation not found: " + conversationId);
    }
}

DTOの @NotNull は「値が未指定でないこと」を確認します。 一方で「そのIDがDBに存在するか」はService層でRepositoryを使って確認します。

9. 変数命名の考え方

sended は英語として不自然です。過去分詞は sent ですが、コードでは無理に過去形にせず、役割を表す名前にすると読みやすくなります。

避けたい名前 推奨名 理由
sendUser messageSender または user メッセージ送信者であることが分かる。
sendedConversation targetConversation または conversation 送信先の会話であることが分かる。
sendedMessage userMessage エンティティ型と役割が一致する。
savedMessage savedUserMessage 保存済みのユーザーメッセージであることが明確。

10. RestClientとOpenAI連携

外部LLMにメッセージを送信して回答を取得する場合、Springの RestClient を使えます。 ChatService にHTTP通信を直接書くより、OpenAiClient のような専用クラスに分けると責務が明確になります。 

sendMessage
1. userId から User を取得
2. conversationId から Conversation を取得
3. UserMessage を保存
4. OpenAiClient で外部LLMへ送信
5. 返ってきた回答を AiMessage として保存
6. AiAnswerResponse を返す

RestClientの基本構成

@Configuration
public class OpenAiConfig {

    @Bean
    RestClient openAiRestClient(
            RestClient.Builder builder,
            @Value("${openai.base-url}") String baseUrl,
            @Value("${openai.api-key}") String apiKey
    ) {
        return builder
                .baseUrl(baseUrl)
                .defaultHeader(HttpHeaders.AUTHORIZATION, "Bearer " + apiKey)
                .defaultHeader(HttpHeaders.CONTENT_TYPE, MediaType.APPLICATION_JSON_VALUE)
                .build();
    }
}

post() の意味

OpenAiResponse response = restClient.post()
        .uri("/responses")
        .body(new OpenAiRequest(model, instructions, userText))
        .retrieve()
        .body(OpenAiResponse.class);
メソッド 意味
post() HTTP POSTリクエストを送る。
uri("/responses") baseUrlに続くパスを指定する。実際には https://api.openai.com/v1/responses へ送る。
body(...) JavaオブジェクトをHTTPリクエスト本文として渡す。JacksonによりJSON化される。
retrieve() リクエストを実行してレスポンスを取得する。
body(OpenAiResponse.class) レスポンスJSONをJavaの OpenAiResponse に変換する。

OpenAIのrequest形式

一旦GPTのみを使うなら、OpenAI Responses APIに対して model、instructions、input を送る形にできます。 instructions は必須ではありませんが、AIの振る舞いを指定したい場合に使います。 

private record OpenAiRequest(
        String model,
        String instructions,
        String input
) {
}

このrecordは次のJSONに変換されます。

{
  "model": "gpt-5.4-mini",
  "instructions": "あなたは日本語で簡潔に回答するAIアシスタントです。",
  "input": "ユーザーの入力文"
}

最小構成にしたい場合は instructions を外し、model と input だけでも構いません。

11. application.properties と ${...}

次の設定を例にします。

openai.base-url=https://api.openai.com/v1
openai.api-key=${OPENAI_API_KEY}
openai.model=${OPENAI_MODEL:gpt-5.4-mini}

左辺の openai.api-key は、Springアプリ内で使う設定名です。 右辺の ${OPENAI_API_KEY} は、OSの環境変数 OPENAI_API_KEY の値を埋め込むという意味です。 

Spring内の設定名       実際の値
openai.api-key   <-  環境変数 OPENAI_API_KEY

APIキーを application.properties に直接書くとGitに混ざる危険があります。 そのため、秘密情報は環境変数から読む形にします。 

// Java側で受け取る
@Value("${openai.model}")
String model

${OPENAI_MODEL:gpt-5.4-mini} はデフォルト値付きです。 環境変数 OPENAI_MODEL があればその値を使い、なければ gpt-5.4-mini を使います。

12. JacksonによるJSON変換

Java recordをJSONに変換する仕組み

RestClient.body(...) にJavaオブジェクトを渡すと、Springの HttpMessageConverter がHTTP本文へ変換します。 JSONの場合は、通常Jacksonが使われます。

このプロジェクトでは spring-boot-starter-webmvc が入っているため、通常はJackson関連の変換が自動構成されます。 そのため、明示的に ObjectMapper を作らなくても、recordをJSONに変換できます。 

private record OpenAiRequest(
        String model,
        String instructions,
        String input
) {
}
.body(new OpenAiRequest(model, instructions, userText))

これは概念的には次のJSONとして送信されます。

{
  "model": "gpt-5.4-mini",
  "instructions": "あなたは日本語で簡潔に回答するAIアシスタントです。",
  "input": "こんにちは"
}

レスポンスの必要な部分だけ受け取れる理由

OpenAIのレスポンスJSONには多くの項目があります。しかし、Java側で必要な項目だけrecordに定義しておけば、 Spring Boot/Jacksonはその項目を中心にマッピングします。 

private record OpenAiResponse(
        List<Output> output
) {
}

JSONに id、model、usage、status などが含まれていても、 Java側に定義していなければ基本的には無視されます。

未知の項目があることは通常問題になりませんが、必要な項目がない場合は null になる可能性があります。 そのため、output == null のようなチェックは必要です。

13. OpenAIレスポンス抽出のStream処理

OpenAIのレスポンスが次のような構造だとします。

{
  "output": [
    {
      "type": "message",
      "content": [
        {
          "type": "output_text",
          "text": "こんにちは。何を手伝いましょうか？"
        }
      ]
    }
  ]
}

この中から回答本文だけを取り出す処理です。

return output.stream()
        .filter(item -> "message".equals(item.type()))
        .filter(item -> item.content() != null)
        .flatMap(item -> item.content().stream())
        .filter(content -> "output_text".equals(content.type()))
        .map(Content::text)
        .filter(text -> text != null && !text.isBlank())
        .findFirst()
        .orElseThrow(() -> new LlmApiException("OpenAI output text is empty."));
処理 意味
output.stream() List<Output> を1件ずつ処理するStreamにする。
filter(item -> "message".equals(item.type())) type が message の出力だけ残す。
filter(item -> item.content() != null) content がnullのものを除外する。
flatMap(item -> item.content().stream()) List<Output> の中にある List<Content> を平らにして、Content を1件ずつ扱えるようにする。
filter(content -> "output_text".equals(content.type())) 回答本文である output_text だけ残す。
map(Content::text) Content オブジェクトから text 文字列だけ取り出す。
findFirst() 条件に合う最初のテキストを取得する。結果は Optional<String>。
orElseThrow(...) テキストが見つからなければ例外を投げる。

for文で書くと、ほぼ次の処理と同じです。

for (Output item : output) {
    if (!"message".equals(item.type())) {
        continue;
    }
    if (item.content() == null) {
        continue;
    }

    for (Content content : item.content()) {
        if (!"output_text".equals(content.type())) {
            continue;
        }
        if (content.text() == null || content.text().isBlank()) {
            continue;
        }

        return content.text();
    }
}

throw new LlmApiException("OpenAI output text is empty.");

14. 推奨コード例

UserMessage

@Entity
@Table(name = "user_messages")
public class UserMessage {
    @Id
    @GeneratedValue(strategy = GenerationType.IDENTITY)
    private Long id;

    @Column(nullable = false, name = "text")
    private String text;

    @ManyToOne(fetch = FetchType.LAZY, optional = false)
    @JoinColumn(nullable = false, name = "user_id")
    private User user;

    @ManyToOne(fetch = FetchType.LAZY, optional = false)
    @JoinColumn(nullable = false, name = "conversation_id")
    private Conversation conversation;

    @Column(name = "created_at", nullable = false)
    private LocalDateTime createdAt;

    protected UserMessage() {
    }

    public UserMessage(String text, User user, Conversation conversation) {
        this.text = text;
        this.user = user;
        this.conversation = conversation;
        this.createdAt = LocalDateTime.now();
    }

    public Long getId() {
        return id;
    }

    public String getText() {
        return text;
    }

    public void editText(String text) {
        this.text = text;
    }

    public User getUser() {
        return user;
    }

    public Conversation getConversation() {
        return conversation;
    }

    public LocalDateTime getCreatedAt() {
        return createdAt;
    }
}

Conversation

@Entity
@Table(name = "conversation")
public class Conversation {
    @Id
    @GeneratedValue(strategy = GenerationType.IDENTITY)
    private Long id;

    @OneToMany(mappedBy = "conversation")
    private List<UserMessage> userMessages = new ArrayList<>();

    @OneToMany(mappedBy = "conversation")
    private List<AiMessage> aiMessages = new ArrayList<>();

    @Enumerated(EnumType.STRING)
    @Column(name = "ai_model", nullable = false)
    private AiModel aiModel;

    @Column(name = "created_at", nullable = false)
    private LocalDateTime createdAt;

    protected Conversation() {
    }

    public Conversation(AiModel aiModel) {
        this.aiModel = aiModel;
        this.createdAt = LocalDateTime.now();
    }

    public Long getId() {
        return id;
    }

    public List<UserMessage> getUserMessages() {
        return userMessages;
    }

    public List<AiMessage> getAiMessages() {
        return aiMessages;
    }

    public AiModel getAiModel() {
        return aiModel;
    }

    public LocalDateTime getCreatedAt() {
        return createdAt;
    }
}

ChatServiceのsendMessage

public AiAnswerResponse sendMessage(SendMessageRequest request) {
    User messageSender = userRepository.findById(request.userId())
            .orElseThrow(() -> new UserNotFoundException(request.userId()));

    Conversation targetConversation = conversationRepository.findById(request.conversationId())
            .orElseThrow(() -> new ConversationNotFoundException(request.conversationId()));

    UserMessage userMessage = new UserMessage(
            request.text(),
            messageSender,
            targetConversation
    );
    userMessageRepository.save(userMessage);

    String answerText = openAiClient.generateAnswer(request.text());

    AiMessage aiMessage = new AiMessage(
            answerText,
            targetConversation.getAiModel(),
            targetConversation
    );
    AiMessage savedAiMessage = aiMessageRepository.save(aiMessage);

    return new AiAnswerResponse(
            savedAiMessage.getId(),
            savedAiMessage.getText(),
            savedAiMessage.getAiModel(),
            savedAiMessage.getCreatedAt()
    );
}

OpenAiClient

@Service
public class OpenAiClient {
    private final RestClient restClient;
    private final String model;

    public OpenAiClient(
            RestClient openAiRestClient,
            @Value("${openai.model}") String model
    ) {
        this.restClient = openAiRestClient;
        this.model = model;
    }

    public String generateAnswer(String userText) {
        OpenAiResponse response = restClient.post()
                .uri("/responses")
                .body(new OpenAiRequest(
                        model,
                        "あなたは日本語で簡潔に回答するAIアシスタントです。",
                        userText
                ))
                .retrieve()
                .body(OpenAiResponse.class);

        if (response == null) {
            throw new LlmApiException("OpenAI response is empty.");
        }

        return response.extractText();
    }

    private record OpenAiRequest(
            String model,
            String instructions,
            String input
    ) {
    }

    private record OpenAiResponse(
            List<Output> output
    ) {
        String extractText() {
            if (output == null) {
                throw new LlmApiException("OpenAI output is empty.");
            }

            return output.stream()
                    .filter(item -> "message".equals(item.type()))
                    .filter(item -> item.content() != null)
                    .flatMap(item -> item.content().stream())
                    .filter(content -> "output_text".equals(content.type()))
                    .map(Content::text)
                    .filter(text -> text != null && !text.isBlank())
                    .findFirst()
                    .orElseThrow(() -> new LlmApiException("OpenAI output text is empty."));
        }
    }

    private record Output(
            String type,
            List<Content> content
    ) {
    }

    private record Content(
            String type,
            String text
    ) {
    }
}

まとめ

  • 外部キーを持つ側には @ManyToOne と @JoinColumn を書く。
  • 親側から子一覧を見たい場合に @OneToMany(mappedBy = "Javaフィールド名") を書く。
  • @Column は単一カラム用であり、関連一覧には付けない。
  • @ManyToOne はデフォルトEAGERなので、通常は fetch = FetchType.LAZY を検討する。
  • JPAエンティティには protected の引数なしコンストラクタが必要。
  • enumの入力制約はDTOの型をenumにし、null制約には @NotNull を使う。
  • 存在しないIDはService層でRepository検索し、404系の独自例外にする。
  • 外部API呼び出しは OpenAiClient に分離すると、Service層が読みやすくなる。
  • application.properties の ${ENV_VAR:default} は環境変数参照とデフォルト値指定。
  • Spring/Jacksonにより、recordはJSONへ、JSONはrecordへ変換できる。
