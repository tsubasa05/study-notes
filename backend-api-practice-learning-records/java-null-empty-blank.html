Java学習ノート: null・空文字・空白文字列

対象コード

private String normalizeBlank(String value) {
  return value == null || value.isBlank() ? null : value.trim();
}

このメソッドは、入力値が未入力扱いであれば null を返し、 値が入っていれば前後の空白を削除して返すための補助メソッドです。

null・空文字・空白文字列の違い

種類 例 意味
null String value = null; 文字列オブジェクト自体が存在しない状態。
空文字 String value = ""; 文字列オブジェクトは存在するが、文字数が 0 の状態。
空白文字列 String value = " "; 文字列オブジェクトは存在し、スペースなどの空白文字だけが入っている状態。

「空」という言葉は曖昧なので、Java の学習では null、""、" " を分けて理解することが重要です。

メソッドの文法

コード 意味
private 同じクラスの中からだけ呼び出せるメソッドであることを表す。
String 戻り値の型。文字列、または null を返す。
normalizeBlank メソッド名。空白扱いの値を正規化する意図がある。
String value 呼び出し元から受け取る文字列の引数。
value == null 値が null かどうかを判定する。
|| OR 条件。左か右のどちらかが true なら全体が true になる。
value.isBlank() 空文字、または空白文字だけの文字列かどうかを判定する。
? null : value.trim() 三項演算子。条件が true なら null、false なら value.trim() を返す。
value.trim() 文字列の前後にある空白を削除する。

処理結果

入力値 判定 戻り値
null value == null が true null
"" value.isBlank() が true null
" " value.isBlank() が true null
" Java " 未入力扱いではない "Java"
"abc" 未入力扱いではない "abc"

この処理を行うメリット

  • 未入力を null に統一できる。
  • DB に NULL、空文字、空白文字列が混在しにくくなる。
  • 後続処理で「値なし」を判定しやすくなる。
  • 検索条件を単純にできる。
  • 画面上は空に見えるのに、プログラム上は値ありと判定されるズレを避けやすい。
  • 入力値の前後に入った不要な空白を保存前に削除できる。

メソッドの意図

このメソッドの主な意図は、「未入力」を表す値を null にそろえ、 意味のある入力値だけを整えた状態で扱えるようにすることです。

つまり、null、空文字、空白文字列を別々に扱うのではなく、 アプリケーション内では「値なし」として統一するための処理です。

if 文で書き換えた場合

三項演算子を使わずに書くと、次のようになります。

private String normalizeBlank(String value) {
  if (value == null || value.isBlank()) {
    return null;
  }

  return value.trim();
}
